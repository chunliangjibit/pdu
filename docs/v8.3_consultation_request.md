# 爆轰物理模拟引擎 V8.3 技术咨询需求书

## 1. 项目概况与 V8.3 架构
PyDetonation-Ultra (PDU) 是一款基于 **JAX 可微分编程** 的高保真爆轰物理数值模拟引擎。目前处于 **V8.3 终极修复版本**。

### V8.3 核心技术栈：
- **状态方程 (EOS)**: 采用 JCZ3 (Exp-6) 模型，支持多组分流体混合逻辑。
- **多相物理模型**: 
  - **固相压缩 (Carbon)**: 模拟石墨向金刚石相变的体积收缩（修正系数 0.645）。
  - **液相膨胀 (Alumina)**: 将爆轰产物中的氧化铝处理为高临界点液相，并考虑液滴的热膨胀效应（修正系数 1.32）。
- **优化框架**: 隐式可微分平衡态求解器 + 等熵线 JWL 拟合（包含 CJ 点声速锚定）。

## 3. 残余误差分析与咨询要点

尽管 V8.3 解决了富碳炸药的重大失真问题，但仍有以下难点需要专家协助：

### 3.1 高能硝胺类爆速偏低问题
- **现象**: HMX 爆速预测值（8448 m/s）较实验值（9110 m/s）仍偏低约 **7.3%**。
- **难点**: 为了平衡混合物压力，我们调整了气体排斥半径，但这似乎牺牲了纯单质产物的声速。
- **咨询重点**: 如何在保证气体势能物理自洽的前提下，进一步通过参数微调或势能形状修正（$\alpha$ 参数标定）补全这 500-600 m/s 的差距？

### 3.2 冷能曲线与凝聚相状态方程
- **现象**: TNT 和 Tritonal 的压力误差仍在 **-10%** 左右。
- **难点**: 目前氧化铝和碳的体积修正采用的是静态系数。
- **咨询重点**: 是否建议引入更加复杂的动态固相 EOS（如 Murnaghan）？对于氧化铝液滴，在高压下的可压缩性是否应采用非线性的温度依赖模型？

### 3.3 多目标标定权重策略
- **需求**: 我们目前同时标定爆速 D、爆压 P 和 JWL 参数。由于不同炸药类别对产物组成的敏感度不同，是否存在针对不同族系（如单质族、含铝族）的权重推荐方案？
